# 情侣日常 App 详细功能需求文档

## 1. 文档说明

本文档面向产品、设计、开发与测试团队，详细定义情侣日常 App 的 8 个核心功能模块需求。每个模块均包含功能描述、用户流程、界面交互、数据需求、边界情况处理，可直接作为开发与验收参考。

## 2. 核心模块总览

本项目核心功能模块如下：

1. 登录与用户资料
2. 情侣绑定与关系主页
3. 聊天互动
4. 纪念日管理
5. 共享相册
6. 心情记录
7. 位置共享
8. 日记与愿望清单

---

## 3. 模块一：登录与用户资料

### 3.1 功能描述

用户可完成注册、登录、自动登录、退出登录，并维护个人资料，包括头像、昵称、签名、生日等信息。该模块是全局访问入口，也是其他业务模块权限校验的前置条件。

### 3.2 用户流程

1. 用户首次打开 App，进入欢迎页/登录页
2. 输入手机号并完成验证码验证登录
3. 首次登录用户进入资料补全页
4. 完成昵称、头像等基础信息设置后进入首页
5. 老用户直接恢复会话进入首页
6. 用户可在“我的”页面编辑个人资料
7. 用户可主动退出登录，返回登录页

### 3.3 界面交互

- 登录页提供手机号输入、验证码获取、验证码输入、登录按钮
- 验证码发送后展示倒计时，防止重复发送
- 首次登录资料补全页需支持头像上传与昵称必填校验
- 个人资料编辑页支持逐项修改并即时反馈保存状态
- 登录中、保存中需展示加载态，防止重复点击

### 3.4 数据需求

- 用户认证信息：手机号、验证码、会话 token
- 用户资料：头像、昵称、生日、签名、通知设置、定位授权状态
- 设备信息：设备 ID、推送注册 ID

### 3.5 边界情况处理

- 验证码发送失败时展示明确错误提示并允许重试
- 验证码失效或错误时不可登录，需重新校验
- 昵称为空时不可完成资料补全
- 自动登录 token 失效时需清理本地会话并跳转登录页
- 用户上传头像失败时允许跳过，后续可再次设置

## 4. 模块二：情侣绑定与关系主页

### 4.1 功能描述

用户可通过邀请码完成情侣关系绑定。绑定成功后，可进入情侣主页查看恋爱天数、伴侣资料、共同纪念日、最近互动等内容，形成应用核心场景入口。

### 4.2 用户流程

1. 用户进入情侣关系页面
2. 选择“生成邀请码”或“输入邀请码”
3. 发起方生成邀请码并分享给对方
4. 接收方输入邀请码并确认绑定
5. 双方确认纪念日等关系信息
6. 绑定成功后进入情侣主页
7. 在主页查看关系摘要、纪念日倒计时、最近动态

### 4.3 界面交互

- 未绑定状态展示空态插画、绑定说明、两个操作入口
- 邀请码展示页需支持复制、分享、刷新
- 输入邀请码页面支持粘贴快捷操作
- 绑定确认弹窗需明确展示双方昵称和纪念日信息
- 情侣主页展示恋爱天数、背景图、伴侣头像、快捷功能入口

### 4.4 数据需求

- 邀请码、邀请码有效期、创建者信息
- 情侣关系状态：待绑定、已绑定、已解除
- 关系资料：纪念日、主页背景图、主题配置、最近互动时间
- 伴侣基础信息：昵称、头像、签名

### 4.5 边界情况处理

- 邀请码过期时提示重新生成
- 同一用户不可重复绑定多个有效情侣关系
- 自己输入自己生成的邀请码时需阻止并提示
- 对方已绑定他人时需提示关系不可建立
- 解除关系后历史数据是否可见需按产品策略处理，默认不物理删除

## 5. 模块三：聊天互动

### 5.1 功能描述

情侣双方可在聊天页面发送文字、图片、位置卡片等内容，并查看发送状态、已读状态，形成高频互动场景。

### 5.2 用户流程

1. 用户进入聊天页
2. 系统加载历史消息并定位到最新记录
3. 用户输入文本后发送消息
4. 用户可选择相册图片或拍照发送
5. 用户可发送当前位置卡片
6. 对方收到推送后进入聊天页查看消息
7. 已读状态同步回发送方

### 5.3 界面交互

- 聊天页底部输入区包含文本框、发送按钮、图片按钮、位置按钮
- 文本为空时发送按钮置灰
- 图片发送前展示预览和压缩中状态
- 消息气泡需区分自己和对方、区分消息类型
- 失败消息需展示重试入口
- 长列表加载历史消息时需平滑分页

### 5.4 数据需求

- 消息内容、消息类型、发送方、接收方、发送时间
- 文件资源信息：图片地址、缩略图、尺寸
- 位置卡片信息：经纬度、地址、静态展示文案
- 消息状态：发送中、发送成功、失败、已读

### 5.5 边界情况处理

- 弱网状态下消息先本地展示为“发送中”
- 发送失败时需允许单条重试，不影响其他消息
- 图片过大时先压缩再上传，压缩失败需中断发送并提示
- 未绑定情侣关系时不可进入聊天页或仅展示引导空态
- 本地时间异常不可影响消息排序，排序应以服务端时间为准

## 6. 模块四：纪念日管理

### 6.1 功能描述

用户可添加、编辑、查看和提醒各种纪念日，包括恋爱纪念日、生日、首次见面日、自定义纪念日等。

### 6.2 用户流程

1. 用户进入纪念日列表页
2. 查看已有纪念日与倒计时
3. 点击新增按钮创建纪念日
4. 设置标题、日期、提醒时间、重复规则
5. 保存后返回列表
6. 到达提醒时间时收到推送通知
7. 用户可编辑或删除纪念日

### 6.3 界面交互

- 列表页展示卡片化纪念日、剩余天数、日期标签
- 新增/编辑页支持日期选择器、提醒开关、重复规则选择
- 重要纪念日可高亮显示
- 删除纪念日需二次确认，防止误操作
- 临近纪念日时首页可展示提醒横幅

### 6.4 数据需求

- 纪念日标题、类型、日期、提醒时间、重复规则、备注
- 创建人信息、关系 ID
- 提醒启用状态与最近提醒记录

### 6.5 边界情况处理

- 日期为空时不可保存
- 重复型纪念日需正确处理跨年倒计时
- 关闭提醒后不再推送，但纪念日仍在列表展示
- 系统通知权限关闭时需提示用户前往系统设置开启
- 同名纪念日允许存在，但应避免默认混淆，可通过类型和日期区分

## 7. 模块五：共享相册

### 7.1 功能描述

用户可上传情侣共同照片，按时间浏览、查看详情、添加文案与地点，形成共同回忆沉淀。

### 7.2 用户流程

1. 用户进入相册页
2. 浏览已上传照片列表
3. 点击上传按钮选择本地照片或拍照
4. 填写照片文案、标签、可见范围
5. 上传成功后展示在时间流中
6. 用户可点击查看大图详情
7. 用户可删除自己上传的照片或管理共享内容

### 7.3 界面交互

- 相册列表支持网格视图与按月分组展示
- 上传入口支持拍照和从相册选择
- 上传前展示裁剪/压缩进度
- 照片详情页展示图片、文案、上传者、拍摄时间、地点
- 空相册状态展示引导文案和上传入口

### 7.4 数据需求

- 原图、缩略图、文案、拍摄时间、拍摄地点、标签
- 上传者、关系 ID、可见范围
- 文件大小、图片尺寸等元数据

### 7.5 边界情况处理

- 上传过程中切到后台时需尽量保持任务状态或提示中断
- 图片格式不支持时需提前拦截
- 单次上传张数过多时需限制并提示
- 文件上传成功但记录创建失败时需做补偿处理或允许稍后修复
- 私密照片仅本人可见，对方浏览时不应出现占位泄露

## 8. 模块六：心情记录

### 8.1 功能描述

用户可记录每日心情，选择心情类型、分值、文案和配图，并决定是否对伴侣可见。该模块用于提升情侣之间的情绪感知与互动。

### 8.2 用户流程

1. 用户进入心情页面
2. 选择当日心情类型与分值
3. 输入文字内容，可选上传图片
4. 选择是否对伴侣可见
5. 提交后生成当日心情记录
6. 对方可在时间流中查看共享心情

### 8.3 界面交互

- 心情录入页提供表情/图标选择、分值滑杆、文本输入区
- 可见性开关需文案明确，如“仅自己可见/伴侣可见”
- 发布成功后展示成功提示，并更新心情时间流
- 时间流可按日期展示双方心情卡片
- 当天已发布时支持再次编辑或追加说明，具体规则可配置

### 8.4 数据需求

- 心情类型、心情分值、文字内容、配图
- 记录日期、发布者、关系 ID
- 可见性状态、创建时间、更新时间

### 8.5 边界情况处理

- 用户未填写文字时允许仅提交心情类型与分值
- 当天重复记录策略需明确：默认允许多条，列表按时间排序
- 对方关闭查看共享内容时，不影响本人记录保存
- 图片上传失败时可降级为纯文字心情
- 涉及敏感文案审核时可预留状态字段用于后续扩展

---

## 9. 模块七：位置共享

### 9.1 功能描述

用户可授权并开启位置共享，查看伴侣最近位置、更新时间和地址信息，也可随时关闭共享状态，强调可控与隐私保护。

### 9.2 用户流程

1. 用户首次进入位置页
2. 系统说明位置用途并请求授权
3. 用户开启位置共享开关
4. App 获取当前定位并上传
5. 双方可查看对方最新位置与更新时间
6. 用户可临时关闭位置共享
7. 再次开启后恢复位置同步

### 9.3 界面交互

- 首次进入展示隐私说明卡片
- 位置页展示地图、伴侣头像标注、地址文本、更新时间
- 共享状态开关需明显，切换时展示说明
- 定位中显示骨架屏或加载态
- 无权限、无网络、GPS 未开启等场景需有明确空态提示

### 9.4 数据需求

- 用户实时经纬度、精度、逆地理地址、城市、更新时间
- 共享状态、权限状态、关系 ID
- 伴侣位置摘要用于卡片展示

### 9.5 边界情况处理

- 用户拒绝定位权限时不得反复强弹，应提供手动开启入口
- 定位失败时保留上次成功定位时间，并提示刷新失败
- 用户关闭共享后，对方仅看到“已关闭共享”状态，不展示旧位置详情
- 未绑定情侣关系时不允许开启共享
- 后台定位频率应可配置，避免明显耗电

## 10. 模块八：日记与愿望清单

### 10.1 功能描述

该模块包含两个子功能：

- 日记：记录个人或共享给伴侣的生活内容
- 愿望清单：记录共同目标、约定计划与完成状态

二者都属于陪伴与沉淀型功能模块。

### 10.2 用户流程

#### 日记流程

1. 用户进入日记页
2. 浏览历史日记列表
3. 点击新增日记
4. 填写标题、正文、图片、天气、心情标签
5. 选择是否共享给伴侣
6. 保存后加入日记列表
7. 可再次编辑或删除本人日记

#### 愿望清单流程

1. 用户进入愿望页
2. 查看待完成、进行中、已完成愿望
3. 点击新增愿望
4. 填写标题、描述、目标日期、分类
5. 保存后进入愿望列表
6. 完成后可修改状态为已完成

### 10.3 界面交互

- 日记列表支持卡片流或时间轴展示
- 日记编辑页支持富文本感的多段输入与图片插入
- 共享日记需有明显标识，方便区分私密内容
- 愿望页可按状态分栏展示，如“待开始/进行中/已完成”
- 愿望卡片展示目标日期、分类标签、进度状态

### 10.4 数据需求

#### 日记

- 标题、正文、图片、天气、心情标签、日期、共享状态
- 作者、关系 ID、创建时间、更新时间

#### 愿望清单

- 标题、描述、分类、目标日期、优先级、状态、完成时间
- 创建人、关系 ID、封面图

### 10.5 边界情况处理

- 日记正文为空时不可保存，避免无意义空内容
- 愿望标题为空时不可创建
- 目标日期早于当前日期时允许保存，但需提示该愿望已逾期
- 私密日记不可被伴侣读取或通过推送摘要泄露
- 已完成愿望再次修改为未完成时，应保留历史状态变更时间用于后续扩展

## 11. 通用需求补充

### 11.1 权限与身份

- 所有核心模块均依赖登录态
- 共享类数据必须校验情侣关系
- 个人私密数据需支持仅自己可见

### 11.2 推送与提醒

- 新消息、纪念日、愿望提醒、共享内容更新可触发推送
- 推送内容仅显示摘要，不泄露完整隐私内容

### 11.3 弱网与异常处理

- 所有提交类操作需具备加载态、失败态、重试机制
- 列表页请求失败应提供重试按钮和空态提示
- 文件上传类功能需支持失败重传

### 11.4 埋点与统计建议

- 登录成功率
- 邀请码生成与绑定成功率
- 日活跃聊天次数
- 相册上传成功率
- 位置共享开启率
- 纪念日提醒点击率

## 12. 验收建议

- 每个模块需覆盖正常流程、异常流程、权限校验流程三类测试用例
- 涉及推送、定位、文件上传的功能应增加真机测试
- 核心数据读写功能需验证情侣双方可见性是否符合预期
